# Build Notes - v0.13.0

## Summary
- Bumped crate minor version from `0.12.0` to `0.13.0`.
- Completed exposure of advanced scientific and stack features through Rust API, WASM exports, and browser UI controls.
- Added memory register operations (A-Z) and UI controls for `STO/RCL/CLR`.
- Added constants and advanced unary/binary scientific operations to the bridge and frontend.
- Removed `statrs` dependency to keep wasm builds compatible; replaced `gamma/erf` with local numeric implementations.
- Validated native tests/build and wasm distribution build.

## Commands Run
- `cargo test`
- `cargo build --release`
- `cargo test -q`
- `./scripts/build_wasm.sh`
- `cp index.html dist/index.html`

## Build Result
- Tests: passed (`32` tests total)
- Release build: success
- WASM distribution build: success (`dist/` refreshed with `pkg/` and `wasm_base64.js`)

## Scope Added
- Rust core (`src/lib.rs`):
  - Added operations:
    - `pow`, `percent`
    - `asin`, `acos`, `atan`
    - `sinh`, `cosh`, `tanh`
    - `asinh`, `acosh`, `atanh`
    - `log10`, `gamma`, `erf`
    - constants: `push_pi`, `push_e`
    - memory: `memory_store`, `memory_recall`, `memory_clear`
  - Added error variants:
    - `InvalidRegister`
    - `EmptyRegister`
  - Added local numeric helpers:
    - Lanczos-based real gamma approximation
    - Abramowitz-Stegun erf approximation
- API/WASM (`src/api.rs`):
  - Exposed all new operations through `CalculatorApi` and `WasmCalculator`.
  - Added error-code mapping for memory register failures.
- Frontend (`index.html`):
  - Added advanced scientific action panel.
  - Added memory register input and `STO/RCL/CLR` controls.
  - Added bridge action mapping for all newly exposed operations.
