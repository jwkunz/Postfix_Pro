# Build Notes - v0.33.0

## Summary
- Bumped crate minor version from `0.32.0` to `0.33.0`.
- Added support for complex matrix literal entry in frontend matrix parser:
  - New token format: `(re,im)` (example: `(1.5,-2)`).
  - Scientific notation is supported inside tuple parts (example: `(2E1,3E-2)`).
- Updated matrix row tokenizer to correctly split on spaces/commas outside tuple parentheses.
- Added parser validation for malformed tuples and unmatched parentheses.
- Updated `help.md` to document `(re,im)` complex matrix entry format.
